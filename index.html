Zero-Dependency Compiler That Outputs Raw Binary

512lang is a tiny, self-contained compiler that transforms source code directly into executable machine code. No LLVM, no external dependencies—just a handcrafted assembler backend in pure C and a frontend parser in Go.

Language Features

Structs & Types

Strong typing with dq (8 bytes), dd (4 bytes), dw (2 bytes), db (1 byte), and ptr (8 bytes). Custom struct definitions.

Control Flow

Complete control structures: if/elif/else, while loops, for loops with proper scoping and nested conditionals.

Pointers & Memory

Full pointer support with address-of operations, dereferencing, and manual memory management primitives.

Arrays & Globals

Global variable declarations, array types, and indexing operations with compile-time bounds checking.

Embedded Assembly

Write inline assembly directly within 512lang code for performance-critical sections or direct hardware access.

Direct Binary Output

Pure C backend assembler that emits raw machine code without intermediate representations or external toolchain dependencies.

Compiler Architecture

Frontend

Go-based lexer, parser, and semantic analyzer. Clean recursive descent parsing with proper error handling.

Backend

Pure C assembler that emits raw machine code directly. No LLVM, no external dependencies.

Output

Direct binary emission with proper ELF headers, symbol tables, and executable segments.
